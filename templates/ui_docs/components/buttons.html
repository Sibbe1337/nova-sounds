Buttons

Buttons provide users with clickable elements that perform actions and navigate between pages. Our button styles match the application's design language and provide clear interactivity cues.

Button Types

Use the appropriate button type based on the action's importance and context.

HTML 
<button class="btn btn-primary">Primary</button>
<button class="btn btn-secondary">Secondary</button>
<button class="btn btn-success">Success</button>
<button class="btn btn-danger">Danger</button>
<button class="btn btn-warning">Warning</button>
<button class="btn btn-info">Info</button>
<button class="btn btn-light">Light</button>
<button class="btn btn-dark">Dark</button>

Outline Buttons

Outline buttons are less emphasized and work well for secondary actions.

HTML 
<button class="btn btn-outline-primary">Primary</button>
<button class="btn btn-outline-secondary">Secondary</button>
<button class="btn btn-outline-success">Success</button>
<button class="btn btn-outline-danger">Danger</button>
<button class="btn btn-outline-warning">Warning</button>
<button class="btn btn-outline-info">Info</button>
<button class="btn btn-outline-light">Light</button>
<button class="btn btn-outline-dark">Dark</button>

Button Sizes

Adjust button size based on its importance and available space.

HTML 
<button class="btn btn-primary btn-lg">Large Button</button>
<button class="btn btn-primary">Default Button</button>
<button class="btn btn-primary btn-sm">Small Button</button>

Button States

Buttons have different states to indicate their interactivity.

HTML 
<button class="btn btn-primary">Default State</button>
<button class="btn btn-primary active">Active State</button>
<button class="btn btn-primary disabled" aria-disabled="true">Disabled State</button>
<button class="btn btn-primary loading">
    <span class="spinner-border spinner-border-sm me-1" role="status" aria-hidden="true"></span>
    Loading...
</button>

Buttons with Icons

Icons enhance button clarity and provide visual cues.

HTML 
<button class="btn btn-primary">
    <i class="fas fa-save me-1"></i> Save
</button>
<button class="btn btn-success">
    <i class="fas fa-check me-1"></i> Confirm
</button>
<button class="btn btn-danger">
    <i class="fas fa-trash me-1"></i> Delete
</button>
<button class="btn btn-info">
    Add <i class="fas fa-plus ms-1"></i>
</button>

Icon-only Buttons

Use icon-only buttons for actions that are universally understood from the icon alone.

HTML 
<button class="btn btn-primary btn-icon" aria-label="Edit">
    <i class="fas fa-edit"></i>
</button>
<button class="btn btn-danger btn-icon" aria-label="Delete">
    <i class="fas fa-trash"></i>
</button>
<button class="btn btn-success btn-icon" aria-label="Add">
    <i class="fas fa-plus"></i>
</button>
<button class="btn btn-secondary btn-icon" aria-label="Search">
    <i class="fas fa-search"></i>
</button>

Button Groups

Group related buttons together for a unified appearance.

HTML 
<div class="btn-group" role="group" aria-label="Basic example">
    <button class="btn btn-primary">Left</button>
    <button class="btn btn-primary">Middle</button>
    <button class="btn btn-primary">Right</button>
</div>

<div class="btn-group" role="group" aria-label="Outline button group">
    <button class="btn btn-outline-secondary">Day</button>
    <button class="btn btn-outline-secondary active">Week</button>
    <button class="btn btn-outline-secondary">Month</button>
</div>

Accessibility Guidelines

  • Always add aria-label to icon-only buttons
  • Use aria-disabled="true" for disabled buttons
  • Add role="group" and aria-label to button groups
  • Ensure sufficient color contrast (4.5:1 ratio) for text
  • Use distinct focus styles for keyboard navigation

Best Practices

Do

  • Use primary buttons for the main action
  • Keep button text concise (1-3 words)
  • Use verb-based labels (e.g., "Save" not "Submit")
  • Add loading states for asynchronous actions
  • Maintain consistent button sizing within a UI

Don't

  • Use too many primary buttons on one page
  • Use generic labels like "Click Here"
  • Mix button styles inconsistently
  • Place buttons where they can be accidentally clicked
  • Use low-contrast colors for button text
